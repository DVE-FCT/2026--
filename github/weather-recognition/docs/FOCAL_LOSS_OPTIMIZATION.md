# Focal Loss 优化实验报告

## 1. 优化背景与目标

### 1.1 问题描述
model_1 使用 CrossEntropyLoss 训练，存在**类别不平衡**问题：
- 简单类别（thunder 95.30%、shine 95.00%）模型已学得很好，损失低但仍占等权重
- 困难类别（cloudy 49.65%、sunny 58.21%）模型难以分辨，被简单样本的噪声平均掉

### 1.2 优化目标
让模型**更关注困难样本**（cloudy、sunny、rainy），降低简单样本（thunder、shine）的权重。

---

## 2. 核心技术方案

### 2.1 Focal Loss 数学原理

标准 CrossEntropyLoss：
$$
L_{CE} = -\log(p_t)
$$

Focal Loss：
$$
L_{FL} = -(1 - p_t)^\gamma \cdot \log(p_t)
$$

核心是 $(1-p_t)^\gamma$ **聚焦因子**：

| 样本类型 | $p_t$（模型置信度） | $(1-p_t)^\gamma$（γ=2） | 最终损失 |
|---------|-----------------|----------------------|--------|
| 简单样本（thunder） | 0.95 | (0.05)² = **0.0025** | 几乎为0 |
| 困难样本（cloudy） | 0.30 | (0.70)² = **0.490** | 保持较大 |

**直观理解**：引入"注意力重分配"机制——让模型**主动放弃"已经学会的简单样本"**，把注意力集中在"还学不会的困难样本"上。

### 2.2 alpha 权重策略

完整公式：
$$
L_{FL} = -\alpha_t \cdot (1-p_t)^\gamma \cdot \log(p_t)
$$

alpha 基于 model_1 每类准确率计算：

| 类别 | model_1 准确率 | alpha 权重 | 类别属性 |
|------|--------------|-----------|---------|
| cloudy | 49.65% | 1.4598 | **困难（高权重）** |
| sunny | 58.21% | 1.2487 | **困难（高权重）** |
| rainy | 64.05% | 1.1366 | **困难（高权重）** |
| haze | 78.31% | 0.9323 | 中等 |
| snow | 76.62% | 0.9526 | 中等 |
| thunder | 95.30% | 0.7678 | 简单（低权重） |
| shine | 95.00% | 0.7702 | 简单（低权重） |
| sunrise | 100% | 0.7321 | 简单（低权重） |

## 3. 实验结果

### 3.1 总体指标对比

| 指标 | model_1 (CE Loss) | model_2 (Focal Loss) | 变化 |
|------|------------------|---------------------|------|
| 测试集准确率 | 73.39% | 72.68% | **-0.71%** |
| 测试损失 | 1.5383 | 1.5448 | +0.43% |

### 3.3 效果分析

**困难类别（优化目标）：**
- ✓ cloudy +2.82%（49.65% → 52.47%）
- ✓✓ sunny +6.79%（58.21% → 65.00%）— 最大受益者
- ✓ rainy +2.62%（64.05% → 66.67%）

**简单类别（非目标）：**
- haze 下降 11.40% 最严重——过度"放弃"了本来能学好的样本
- shine 下降 5.87%，thunder 下降 2.58%

## 4. 结论与建议

### 4.1 结论

| 评估项 | 结果 |
|--------|------|
| 优化方向是否正确 | ✓ 是——困难类别确实改善 |
| 总体准确率 | ✗ 略降（-0.71%），收益被简单类别退化抵消 |
| 副作用 | gamma=2 过激，简单类别衰减过度 |

### 4.2 改进建议

| 方案 | 说明 | 推荐程度 |
|------|------|---------|
| 降低 gamma 至 1.0~1.5 | 减轻简单样本衰减，缓解 haze/shine 下降 | ★★★ |
| 调整 alpha 权重差异 | 使用更温和的权重方案，不过度打压简单类别 | ★★ |
| 结合数据增强 | 困难类别可能更多需要增强而非损失重加权 | ★★ |
| Label Smoothing | 引入标签平滑正则化，避免过度自信 | ★ |

### 4.3 下一步计划

1. 降低 gamma 至 1.5 进行对比实验
2. 观察困难类别提升是否保持，简单类别是否回暖
3. 如效果平衡，再进行分支提交与合并
